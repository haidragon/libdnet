# libdnet 编译使用指南

## 环境准备

### Linux/macOS/WSL

#### 1. 安装编译依赖

```bash
# Ubuntu/Debian
sudo apt-get install build-essential automake autoconf libtool

# CentOS/RHEL
sudo yum groupinstall "Development Tools"
sudo yum install automake autoconf libtool

# macOS (使用 Homebrew)
brew install autoconf automake libtool
```

#### 2. 可选依赖

```bash
# Python 绑定（可选）
sudo apt-get install python3-dev

# Check 测试框架（可选）
sudo apt-get install check
```

### Windows

#### 使用 MSYS2 + MinGW（推荐）

```bash
# 1. 下载安装 MSYS2: https://www.msys2.org/

# 2. 在 MSYS2 终端安装工具链
pacman -S mingw-w64-x86_64-gcc mingw-w64-x86_64-autotools mingw-w64-x86_64-libtool

# 3. 必须先编译 libpcap（WinPcap 的开源版本）
git clone https://github.com/the-tcpdump-group/libpcap.git
cd libpcap
cmake -G "MSYS Makefiles" -DCMAKE_INSTALL_PREFIX=/usr/local .
make
make install
```

#### 预编译依赖

- **WinPcap**: http://winpcap.polito.it/install/default.htm
- **PktFilter**: http://www.hsc.fr/ressources/outils/pktfilter/index.html.en

## 编译安装 libdnet

### 标准 autotools 编译（推荐）

```bash
# 进入源码目录
cd libdnet-1.13

# 1. 生成 configure 脚本（如果 configure 文件不存在）
autoreconf -i

# 2. 配置编译选项
./configure \
  --prefix=/usr/local \              # 安装路径
  --with-python=/usr/bin/python3 \    # Python 绑定（可选）
  --with-check=/usr \                # Check 测试框架（可选）

# 3. 编译
make

# 4. 运行测试（可选）
make check

# 5. 安装（需要 root 权限）
sudo make install

# 6. 刷新动态库缓存
sudo ldconfig
```

### 配置选项说明

| 选项 | 说明 |
|------|------|
| `--prefix=PATH` | 指定安装路径，默认 `/usr/local` |
| `--with-python=PATH` | 指定 Python 解释器路径 |
| `--with-check=PATH` | 指定 Check 测试框架路径 |
| `--disable-shared` | 只编译静态库 |
| `--disable-static` | 只编译动态库 |
| `--enable-debug` | 启用调试符号 |

### 只编译不安装

```bash
# 仅编译，不安装
./configure && make

# 库文件位置: src/.libs/libdnet.so (动态库)
#              src/.libs/libdnet.a  (静态库)
```

## 编译测试程序

### Linux/macOS/WSL

#### 方法 1: 使用已安装的库（推荐）

```bash
# 确保 libdnet 已安装
ldconfig -p | grep dnet

# 编译
gcc main.c -ldnet -o test

# 运行（需要 root 权限）
sudo ./test
```

#### 方法 2: 使用源码目录中的库

```bash
# 动态链接
gcc main.c \
  -I../../include \
  -L../../src/.libs \
  -ldnet \
  -o test

# 设置库路径并运行
export LD_LIBRARY_PATH=../../src/.libs:$LD_LIBRARY_PATH
sudo ./test
```

#### 方法 3: 静态链接（无需运行时库）

```bash
gcc main.c \
  -I../../include \
  -L../../src/.libs \
  -ldnet \
  -static \
  -o test_static

sudo ./test_static
```

### Windows (MinGW)

```bash
# 动态链接
gcc main.c -I../../include -L../../src/.libs -ldnet -lwsock32 -o test.exe

# 静态链接
gcc main.c -I../../include ../../src/.libs/libdnet.a -lwsock32 -static -o test.exe

# 注意：需要以管理员身份运行
test.exe
```

## 常见问题

### 1. 找不到共享库

```
error while loading shared libraries: libdnet.so.1: cannot open shared object file
```

**解决方案：**

```bash
# 方案 A: 临时设置库路径
export LD_LIBRARY_PATH=/path/to/libdnet/src/.libs:$LD_LIBRARY_PATH

# 方案 B: 安装到系统路径
sudo make install
sudo ldconfig

# 方案 C: 静态链接
gcc main.c -I../../include -L../../src/.libs -ldnet -static -o test
```

### 2. 编译报错找不到头文件

```
fatal error: dnet.h: No such file or directory
```

**解决方案：**

```bash
# 添加 include 路径
gcc main.c -I/path/to/libdnet/include -ldnet -o test
```

### 3. ICMP 头文件兼容性问题

```
error: expected ':', ',', ';', '}' or '__attribute__' before '.' token
```

**解决方案：**

这是因为 libdnet 的 `icmp.h` 使用 C99 可变数组。避免直接包含：

```c
// 正确做法
#include <dnet.h>

// 不要单独包含
// #include <dnet/icmp.h>  // ❌ 可能有问题

// 或者自定义 ICMP 结构体（见 main.c 示例）
```

### 4. 权限不足

```
intf_open: Permission denied
ip_send: Operation not permitted
```

**解决方案：**

```bash
# 需要使用 sudo 运行
sudo ./test

# 或者设置 capabilities（替代 sudo）
sudo setcap cap_net_raw,cap_net_admin=eip ./test
```

### 5. Windows 编译错误

```
undefined reference to `GetAdaptersAddresses'
```

**解决方案：**

```bash
# 添加 ws2_32 和 iphlpapi 库
gcc main.c -ldnet -lws2_32 -liphlpapi -o test.exe
```

## 开发示例

### 查看网络接口

```c
#include <dnet.h>

int main() {
    intf_t *intf;
    struct intf_entry entry;

    if ((intf = intf_open()) == NULL) {
        perror("intf_open");
        return 1;
    }

    entry.intf_len = sizeof(entry);
    while (intf_loop(intf, &entry, NULL) == 0) {
        printf("%s: %s\n", entry.intf_name, addr_ntoa(&entry.intf_addr));
        entry.intf_len = sizeof(entry);
    }

    intf_close(intf);
    return 0;
}
```

### 发送 ICMP 包

```bash
# 编译
gcc icmp_test.c -ldnet -o icmp_test

# 运行（需要 root）
sudo ./icmp_test
```

### 查看路由表

```c
#include <dnet.h>

int main() {
    route_t *route;
    struct route_entry entry;

    if ((route = route_open()) == NULL) {
        perror("route_open");
        return 1;
    }

    while (route_loop(route, &entry, NULL) == 0) {
        printf("%s via %s\n",
               addr_ntoa(&entry.route_dst),
               addr_ntoa(&entry.route_gw));
    }

    route_close(route);
    return 0;
}
```

## 编译脚本

### Linux 编译脚本

创建 `build.sh`:

```bash
#!/bin/bash

# 编译脚本
set -e

DNET_ROOT="../../src/.libs"
INCLUDE="../../include"

# 编译
gcc main.c \
    -I$INCLUDE \
    -L$DNET_ROOT \
    -ldnet \
    -o test

echo "编译完成！"
echo "运行: export LD_LIBRARY_PATH=$DNET_ROOT:\$LD_LIBRARY_PATH && sudo ./test"
```

使用:

```bash
chmod +x build.sh
./build.sh
```

### Windows 编译脚本

创建 `build.bat`:

```batch
@echo off
gcc main.c -I..\..\include -L..\..\src\.libs -ldnet -lws2_32 -o test.exe
echo 编译完成！
echo 请以管理员身份运行 test.exe
```
