## 1. 校验和基础概念

### 1.1 校验和的作用

校验和在网络协议中起着至关重要的作用：

- **数据完整性验证**：确保数据在传输过程中未被篡改
- **传输错误检测**：发现链路层的比特错误
- **协议兼容性**：符合RFC标准的网络协议实现

### 1.2 网络协议中的校验和类型

| 协议 | 校验和类型 | RFC标准 | 覆盖范围 |
|------|-----------|---------|----------|
| IPv4 | Internet Checksum | RFC 791 | IP头部 |
| IPv6 | 无（由上层协议保证） | RFC 2460 | - |
| TCP | Internet Checksum | RFC 793 | 伪头部+TCP头部+数据 |
| UDP | Internet Checksum | RFC 768 | 伪头部+UDP头部+数据 |
| ICMP | Internet Checksum | RFC 792 | ICMP头部+数据 |
| ICMPv6 | Internet Checksum | RFC 4443 | 伪头部+ICMPv6头部+数据 |
| SCTP | CRC32-C | RFC 4960 | SCTP头部+数据 |

### 1.3 Internet校验和算法原理

Internet校验和是一种**16位反码和**校验算法：

1. **计算过程**：
   - 将数据视为16位字序列
   - 对所有16位字求和
   - 将进位加到低16位（折叠）
   - 取反码作为最终校验和

2. **验证过程**：
   - 对接收的数据（包含校验和字段）执行相同计算
   - 如果结果为0xFFFF，则校验通过

3. **特点**：
   - 简单高效
   - 字节序无关
   - 适合硬件实现
   - 对突发错误检测效果好

### 1.4 CRC32-C算法原理

CRC32-C（Castagnoli）是SCTP协议使用的校验和算法：

1. **多项式**：0x1EDC6F41
2. **初始值**：0xFFFFFFFF
3. **最终异或**：0xFFFFFFFF
4. **输出转换**：字节序翻转

特点：
- 比Internet校验和更强的错误检测能力
- 更适合数据完整性保护
- 现代CPU通常有硬件加速支持（SSE4.2）

## 2. libdnet校验和架构设计

### 2.1 核心设计原则

libdnet的校验和计算遵循以下设计原则：

1. **零拷贝设计**：直接在数据包缓冲区上计算
2. **自动协议识别**：根据IP协议字段自动选择校验和算法
3. **跨平台一致性**：所有平台产生相同的校验和结果
4. **高性能优化**：使用循环展开和快速路径
5. **完整性覆盖**：支持所有需要校验和的协议

```c
    tcp->th_sum = 0;
    sum = ip_cksum_add(tcp, len, 0) + htonl(ip->ip_p + len);
```

### 2.5 宏定义

```c
/* include/dnet/ip.h:453-454 */
#define ip_cksum_carry(x) \
    (x = (x >> 16) + (x & 0xffff), (~(x + (x >> 16)) & 0xffff))
```

这个宏实现了：
1. 将进位折叠到低16位
2. 再次折叠可能的进位
3. 取反码得到最终校验和

## 3. IP头部校验和计算

### 3.1 IP头部校验和规范

根据RFC 791，IPv4头部校验和的特点：

- **计算范围**：仅计算IP头部（包括选项）
- **校验和字段**：计算时先置0
- **分片处理**：只有第一个分片计算校验和
- **重新计算**：每次TTL减少都需要重新计算

### 3.2 源码分析

#### 3.2.1 ip_checksum()函数

**文件位置**：`src/ip-util.c:132-182`

```c
void ip_checksum(void *buf, size_t len, int flags)
{
    struct ip_hdr *ip;
    int hl, off, sum;

    if (len < IP_HDR_LEN)
        return;

    ip = (struct ip_hdr *)buf;
    hl = ip->ip_hl << 2;          /* 头部长度（字节） */
    ip->ip_sum = 0;                /* 清零校验和字段 */
    sum = ip_cksum_add(ip, hl, 0); /* 计算头部校验和 */
    ip->ip_sum = ip_cksum_carry(sum); /* 存储结果 */

    /* 如果设置了IP_CHECKSUM_FRAGMENT标志，只计算IP头部 */
    if (flags & IP_CHECKSUM_FRAGMENT)
        return;

    off = htons(ip->ip_off);

    /* 如果是分片（除了最后一个分片），不计算上层协议校验和 */
    if ((off & IP_OFFMASK) != 0 || (off & IP_MF) != 0)
        return;

    len -= hl;  /* 剩余数据长度 */

    /* 根据IP协议类型计算上层协议校验和 */
    if (ip->ip_p == IP_PROTO_TCP) {
        struct tcp_hdr *tcp = (struct tcp_hdr *)((u_char *)ip + hl);
        if (len >= TCP_HDR_LEN) {
            tcp_checksum(ip, tcp, len);
        }
    } else if (ip->ip_p == IP_PROTO_UDP) {
        struct udp_hdr *udp = (struct udp_hdr *)((u_char *)ip + hl);
        if (len >= UDP_HDR_LEN) {
            udp_checksum(ip, udp, len);
        }
    } else if (ip->ip_p == IP_PROTO_ICMP || ip->ip_p == IP_PROTO_IGMP) {
        struct icmp_hdr *icmp = (struct icmp_hdr *)((u_char *)ip + hl);
        if (len >= ICMP_HDR_LEN) {
            icmp_checksum(icmp, len);
        }
    } else if (ip->ip_p == IP_PROTO_SCTP) {
        struct sctp_hdr *sctp = (struct sctp_hdr *)((u_char *)ip + hl);
        if (len >= SCTP_HDR_LEN) {
            sctp->sh_sum = 0;
            sctp->sh_sum = htonl(_crc32c((u_char *)sctp, len));
        }
    }
}
```

#### 3.2.2 关键设计点

1. **分片处理**：
   ```c
   if ((off & IP_OFFMASK) != 0 || (off & IP_MF) != 0)
       return;
   ```
   - 只对完整的或最后一个分片计算上层校验和
   - 中间分片不计算（因为TCP/UDP/ICMP头部在第一个分片）

2. **协议分发**：
   - TCP：计算伪头部校验和
   - UDP：计算伪头部校验和（特殊处理0值）
   - ICMP/IGMP：计算简单校验和
   - SCTP：使用CRC32-C算法

3. **长度检查**：
   ```c
   if (len >= TCP_HDR_LEN)  /* 确保有足够的头部数据 */
   ```

### 3.3 计算流程图

```
开始
  │
  ├─→ 检查长度 >= IP_HDR_LEN
  │
  ├─→ 提取IP头部信息
  │     ├─→ 头部长度 hl = ip_hl << 2
  │     └─→ 标志位 off = ip_off
  │
  ├─→ 清零 ip_sum
  │
  ├─→ 计算IP头部校验和
  │     ├─→ ip_cksum_add(ip, hl, 0)
  │     └─→ ip_cksum_carry(sum)
  │
  ├─→ 检查IP_CHECKSUM_FRAGMENT标志
  │     └─→ 如果设置，直接返回
  │
  ├─→ 检查是否为分片
  │     └─→ 如果是中间分片，返回
  │
  ├─→ 根据协议类型分发
  │     ├─→ IP_PROTO_TCP → tcp_checksum()
  │     ├─→ IP_PROTO_UDP → udp_checksum()
  │     ├─→ IP_PROTO_ICMP/IGMP → icmp_checksum()
  │     └─→ IP_PROTO_SCTP → _crc32c()
  │
  └─→ 结束
```

### 3.4 IP_CHECKSUM_FRAGMENT标志

```c
/* include/dnet/ip.h:449 */
#define IP_CHECKSUM_FRAGMENT 1
```

用途：
- 手动分片时，对每个分片只计算IP头部校验和
- 避免错误地尝试计算不完整的上层协议校验和

示例：
```c
/* 发送第一个分片 */
ip_checksum(buf1, len1, 0);  /* 计算IP和上层协议校验和 */

/* 发送后续分片 */
ip_checksum(buf2, len2, IP_CHECKSUM_FRAGMENT);  /* 只计算IP头部 */
```

## 4. TCP校验和计算

### 4.1 TCP伪头部

TCP校验和需要包含**伪头部**（Pseudo-header），以确保包含IP层信息：

```
+--------+--------+--------+--------+
|           源IP地址（4字节）         |
+--------+--------+--------+--------+
|         目的IP地址（4字节）         |
+--------+--------+--------+--------+
|  保留   |  协议   |     TCP长度     |
+--------+--------+--------+--------+
|         TCP头部 + 数据          |
+--------+--------+--------+--------+
```

### 4.2 源码分析

#### 4.2.1 tcp_checksum()函数

**文件位置**：`src/ip-util.c:102-109`

```c
void tcp_checksum(struct ip_hdr *ip, struct tcp_hdr *tcp, size_t len)
{
    int sum;
    tcp->th_sum = 0;  /* 清零校验和字段 */
    
    /* 1. 计算TCP头部+数据的校验和 */
    sum = ip_cksum_add(tcp, len, 0);
    
    /* 2. 加上伪头部信息（协议+长度） */
    sum = sum + htonl(ip->ip_p + len);
    
    /* 3. 加上源和目的IP地址（8字节） */
    sum = ip_cksum_add(&ip->ip_src, 8, sum);
    
    /* 4. 进位折叠并取反码 */
    tcp->th_sum = ip_cksum_carry(sum);
}
```

#### 4.2.2 关键点解析

1. **协议长度编码**：
   ```c
   sum = sum + htonl(ip->ip_p + len);
   ```
   - `ip->ip_p`：协议号（TCP = 6）
   - `len`：TCP头部+数据长度
   - 使用`htonl()`确保大端序

2. **IP地址处理**：
   ```c
   sum = ip_cksum_add(&ip->ip_src, 8, sum);
   ```
   - 直接累加8字节（源IP + 目的IP）
   - `ip_cksum_add`会自动处理字节序

3. **验证接收时的校验和**：
   ```c
   /* 验证TCP校验和 */
   tcp->th_sum = 0;  /* 先清零 */
   sum = ip_cksum_add(tcp, len, 0);
   sum = ip_cksum_add(&ip->ip_src, 8, sum) + htonl(ip->ip_p + len);
   cksum = ip_cksum_carry(sum);
   
   /* 如果cksum == 0xFFFF，校验通过 */
   if (cksum != 0xFFFF) {
       /* 校验失败 */
   }
   ```

### 4.3 计算示例

假设发送一个TCP SYN包：

```c
#include <dnet.h>
#include <string.h>

void send_tcp_syn(void) {
    u_char packet[128];
    struct ip_hdr *ip = (struct ip_hdr *)packet;
    struct tcp_hdr *tcp = (struct tcp_hdr *)(packet + 20);
    
    /* 构造IP头部 */
    memset(ip, 0, sizeof(*ip));
    ip->ip_v = 4;
    ip->ip_hl = 5;
    ip->ip_tos = 0;
    ip->ip_len = htons(40);  /* IP头20 + TCP头20 */
    ip->ip_id = rand() & 0xffff;
    ip->ip_off = 0;
    ip->ip_ttl = 64;
    ip->ip_p = IP_PROTO_TCP;
    ip->ip_sum = 0;
    ip->ip_src = inet_addr("192.168.1.100");
    ip->ip_dst = inet_addr("192.168.1.200");
    
    /* 构造TCP头部 */
    memset(tcp, 0, sizeof(*tcp));
    tcp->th_sport = htons(12345);
    tcp->th_dport = htons(80);
    tcp->th_seq = htonl(1000);
    tcp->th_ack = 0;
    tcp->th_off = 5;
    tcp->th_flags = TH_SYN;
    tcp->th_win = htons(65535);
    tcp->th_urp = 0;
    tcp->th_sum = 0;

    /* 发送数据包... */

```

### 4.4 常见错误

1. **忘记清零校验和字段**：
   ```c
   /* 错误 */
   sum = ip_cksum_add(tcp, len, 0);  /* tcp->th_sum未清零 */
   
   /* 正确 */
   tcp->th_sum = 0;
   sum = ip_cksum_add(tcp, len, 0);
   ```

2. **长度错误**：
   ```c
   /* 错误：只传TCP头部 */
   tcp_checksum(ip, tcp, TCP_HDR_LEN);
   
   /* 正确：包括数据 */
   tcp_checksum(ip, tcp, TCP_HDR_LEN + data_len);
   ```

3. **字节序错误**：
   ```c
   /* 错误：未转换字节序 */
   sum = sum + ip->ip_p + len;
   
   /* 正确：使用htonl */
   sum = sum + htonl(ip->ip_p + len);
   ```

## 5. UDP校验和计算

### 5.1 UDP校验和特点

UDP校验和与TCP类似，但有一个重要特性：

- **可选性**：UDP校验和是可选的（RFC 768）
- **零值特殊处理**：如果校验和为0，表示发送方禁用校验和
- **零值替换**：计算结果为0时，必须替换为0xFFFF

### 5.2 源码分析

#### 5.2.1 udp_checksum()函数

**文件位置**：`src/ip-util.c:112-121`

```c
void udp_checksum(struct ip_hdr *ip, struct udp_hdr *udp, size_t len)
{
    int sum;
    udp->uh_sum = 0;  /* 清零校验和字段 */
    
    /* 1. 计算UDP头部+数据的校验和 */
    sum = ip_cksum_add(udp, len, 0);
    
    /* 2. 加上伪头部信息（协议+长度） */
    sum = sum + htonl(IP_PROTO_UDP + len);
    
    /* 3. 加上源和目的IP地址（8字节） */
    sum = ip_cksum_add(&ip->ip_src, 8, sum);
    
    /* 4. 进位折叠并取反码 */
    udp->uh_sum = ip_cksum_carry(sum);
    
    /* 5. 特殊处理：如果结果为0，替换为0xFFFF */
    if (!udp->uh_sum)
        udp->uh_sum = 0xffff;  /* RFC 768 */
}
```

#### 5.2.2 与TCP的区别

| 特性 | TCP | UDP |
|------|-----|-----|
| 伪头部 | 相同 | 相同 |
| 必填性 | 必须计算 | 可选（为0表示禁用） |
| 零值处理 | 允许 | 替换为0xFFFF |
| 协议号 | IP_PROTO_TCP (6) | IP_PROTO_UDP (17) |

### 5.3 禁用UDP校验和

有些应用为了性能会禁用UDP校验和：

```c
/* 方法1：直接设置为0 */
udp->uh_sum = 0;  /* 接收方不验证 */

/* 方法2：不调用udp_checksum() */
/* 注意：某些网络设备可能会在传输时添加校验和 */
```

### 5.4 验证UDP校验和

```c
/* 接收端验证 */
int verify_udp_checksum(struct ip_hdr *ip, struct udp_hdr *udp, size_t len) {
    uint16_t received_sum = udp->uh_sum;
    
    /* 如果校验和为0，表示发送方禁用了校验和 */
    if (received_sum == 0) {
        return 1;  /* 跳过验证 */
    }
    
    /* 重新计算校验和 */
    udp->uh_sum = 0;
    int sum = ip_cksum_add(udp, len, 0);
    sum = sum + htonl(IP_PROTO_UDP + len);
    sum = ip_cksum_add(&ip->ip_src, 8, sum);
    uint16_t calc_sum = ip_cksum_carry(sum);
    
    /* 恢复原始值 */
    udp->uh_sum = received_sum;
    
    /* 检查是否匹配 */
    return calc_sum == 0xffff;
}
```

## 6. ICMP校验和计算

### 6.1 ICMP校验和特点

ICMP校验和的特点：

- **简单校验**：只覆盖ICMP头部+数据
- **无伪头部**：不需要IP地址信息
- **全包覆盖**：包括所有ICMP类型和代码

### 6.2 源码分析

#### 6.2.1 icmp_checksum()函数

**文件位置**：`src/ip-util.c:124-129`

```c
void icmp_checksum(struct icmp_hdr *icmp, size_t len)
{
    int sum;
    
    /* 计算ICMP头部+数据的校验和 */
    sum = ip_cksum_add(icmp, len, 0);
    
    /* 进位折叠并取反码 */
    icmp->icmp_cksum = ip_cksum_carry(sum);
}
```

#### 6.2.2 ICMP Echo示例

```c
#include <dnet.h>

void send_icmp_echo(void) {
    u_char packet[128];
    struct ip_hdr *ip = (struct ip_hdr *)packet;
    struct icmp_hdr *icmp = (struct icmp_hdr *)(packet + 20);
    struct icmp_msg_echo *echo = (struct icmp_msg_echo *)(packet + 24);
    
    /* 构造IP头部 */
    memset(ip, 0, sizeof(*ip));
    ip->ip_v = 4;
    ip->ip_hl = 5;
    ip->ip_len = htons(28);  /* IP头20 + ICMP头4 + 数据8 */
    ip->ip_id = rand() & 0xffff;
    ip->ip_ttl = 64;
    ip->ip_p = IP_PROTO_ICMP;
    ip->ip_src = inet_addr("192.168.1.100");
    ip->ip_dst = inet_addr("8.8.8.8");
    
    /* 构造ICMP Echo请求 */
    icmp->icmp_type = ICMP_ECHO;
    icmp->icmp_code = 0;
    icmp->icmp_cksum = 0;
    
    /* Echo数据 */
    echo->icmp_id = htons(12345);
    echo->icmp_seq = htons(1);
    memcpy(echo->icmp_data, "Hello", 5);
    
    /* 计算校验和 */
    icmp_checksum(icmp, 12);  /* ICMP头4 + 数据8 */
```

## 7. ICMPv6校验和计算

### 7.1 ICMPv6伪头部

ICMPv6需要IPv6伪头部：

```
+--------+--------+--------+--------+
|           源IPv6地址（16字节）         |
+--------+--------+--------+--------+
|         目的IPv6地址（16字节）         |
+--------+--------+--------+--------+
|         上层包长度（4字节）            |
+--------+--------+--------+--------+
|            保留（3字节）              |
+--------+--------+--------+
|          下一个头（1字节）            |
+--------+--------+--------+--------+
|          ICMPv6头部+数据             |
+--------+--------+--------+--------+
```

### 7.2 源码分析

#### 7.2.1 ip6_checksum()函数

**文件位置**：`src/ip6.c:18-72`

```c
void ip6_checksum(void *buf, size_t len)
{
    struct ip6_hdr *ip6 = (struct ip6_hdr *)buf;
    struct ip6_ext_hdr *ext;
    u_char *p, nxt;
    int i, sum;

    nxt = ip6->ip6_nxt;  /* 获取下一个头部类型 */

    /* 跳过扩展头部，找到上层协议头部 */
    for (i = IP6_HDR_LEN; IP6_IS_EXT(nxt); i += (ext->ext_len + 1) << 3) {
        if (i >= (int)len) return;
        ext = (struct ip6_ext_hdr *)((u_char *)buf + i);
        nxt = ext->ext_nxt;
    }
    
    p = (u_char *)buf + i;  /* 上层协议头部位置 */
    len -= i;               /* 上层协议数据长度 */

    if (nxt == IP_PROTO_TCP) {
        struct tcp_hdr *tcp = (struct tcp_hdr *)p;

        if (len >= TCP_HDR_LEN) {
            tcp->th_sum = 0;
            /* 计算TCP数据 + 协议号(8) + 长度 */
            sum = ip_cksum_add(tcp, len, 0) + htons(nxt + (u_short)len);
            /* 加上IPv6伪头部（32字节） */
            sum = ip_cksum_add(&ip6->ip6_src, 32, sum);
            tcp->th_sum = ip_cksum_carry(sum);
        }
    } else if (nxt == IP_PROTO_UDP) {
        struct udp_hdr *udp = (struct udp_hdr *)p;

        if (len >= UDP_HDR_LEN) {
            udp->uh_sum = 0;
            sum = ip_cksum_add(udp, len, 0) + htons(nxt + (u_short)len);
            sum = ip_cksum_add(&ip6->ip6_src, 32, sum);
            /* UDP特殊处理：0替换为0xFFFF */
            if ((udp->uh_sum = ip_cksum_carry(sum)) == 0)
                udp->uh_sum = 0xffff;
        }
    } else if (nxt == IP_PROTO_ICMPV6) {
        struct icmp_hdr *icmp = (struct icmp_hdr *)p;

        if (len >= ICMP_HDR_LEN) {
            icmp->icmp_cksum = 0;
            sum = ip_cksum_add(icmp, len, 0) + htons(nxt + (u_short)len);
            sum = ip_cksum_add(&ip6->ip6_src, 32, sum);
            icmp->icmp_cksum = ip_cksum_carry(sum);
        }
    } else if (nxt == IP_PROTO_ICMP || nxt == IP_PROTO_IGMP) {
        /* 注意：这里处理的是IPv4封装的ICMP */
        struct icmp_hdr *icmp = (struct icmp_hdr *)p;

        if (len >= ICMP_HDR_LEN) {
            icmp->icmp_cksum = 0;
            /* 没有伪头部 */
            sum = ip_cksum_add(icmp, len, 0);
            icmp->icmp_cksum = ip_cksum_carry(sum);
        }
    }
}
```

#### 7.2.2 关键点

1. **扩展头部处理**：
   ```c
   #define IP6_IS_EXT(n) \
       ((n) == IP_PROTO_HOPOPTS || (n) == IP_PROTO_DSTOPTS || \
        (n) == IP_PROTO_ROUTING || (n) == IP_PROTO_FRAGMENT)
   ```
   - 跳过所有扩展头部
   - 找到真正的上层协议头部

2. **伪头部差异**：
   - IPv4：8字节（源IP + 目的IP）
   - IPv6：32字节（源IPv6 + 目的IPv6 + 保留 + 下一个头）

3. **协议号编码**：
   ```c
   sum = sum + htons(nxt + (u_short)len);
   ```
   - 使用`htons`而非`htonl`（因为协议号是1字节）

**文件位置**：`src/ip-util.c:18-38`

```c
static unsigned long _crc32c(unsigned char *buf, int len)
{
    int i;
    unsigned long crc32 = ~0L;  /* 初始值：全1 */
    unsigned long result;
    unsigned char byte0, byte1, byte2, byte3;

    /* 对每个字节应用CRC32C */
    for (i = 0; i < len; i++) {
        CRC32C(crc32, buf[i]);
    }

    result = ~crc32;  /* 最终异或 */

    /* 字节序翻转（输出转换） */
    byte0 =  result        & 0xff;
    byte1 = (result >>  8) & 0xff;
    byte2 = (result >> 16) & 0xff;
    byte3 = (result >> 24) & 0xff;
    crc32 = ((byte0 << 24) | (byte1 << 16) | (byte2 <<  8) | byte3);
    
    return crc32;
}
```

## 10. 校验和算法详解

### 10.1 ip_cksum_add()函数

**文件位置**：`src/ip-util.c:185-238`

```c
int ip_cksum_add(const void *buf, size_t len, int cksum)
{
    uint16_t *sp = (uint16_t *)buf;
    int n, sn;

    /* 处理奇数长度的特殊情况 */
    if (len == 1) {
        const uint8_t *b = buf;
        return htons(((uint16_t)*b) << 8);
    }

    sn = len / 2;  /* 16位字的数量 */
    n = (sn + 15) / 16;  /* 循环次数（每16个字一次） */

    /* 使用Duff's Device展开循环 */
    switch (sn % 16) {
    case 0: do {
        cksum += *sp++;
    case 15:
        cksum += *sp++;
    case 14:
        cksum += *sp++;
    case 13:
        cksum += *sp++;
    case 12:
        cksum += *sp++;
    case 11:
        cksum += *sp++;
    case 10:
        cksum += *sp++;
    case 9:
        cksum += *sp++;
    case 8:
        cksum += *sp++;
    case 7:
        cksum += *sp++;
    case 6:
        cksum += *sp++;
    case 5:
        cksum += *sp++;
    case 4:
        cksum += *sp++;
    case 3:
        cksum += *sp++;
    case 2:
        cksum += *sp++;
    case 1:
        cksum += *sp++;
    } while (--n > 0);
    }
    
    /* 处理奇数长度 */
    if (len & 1)
        cksum += htons(*(u_char *)sp << 8);

    return (cksum);
}
```

### 10.2 Duff's Device优化

Duff's Device是一种循环展开技术：

```c
switch (sn % 16) {
case 0: do {
    cksum += *sp++;
case 15: cksum += *sp++;
case 14: cksum += *sp++;
/* ... */
case 1:  cksum += *sp++;
} while (--n > 0);
}
```

优点：
- 减少循环开销
- 适合处理16的倍数的数据
- 兼容所有C编译器

### 10.3 ip_cksum_carry()宏

**文件位置**：`include/dnet/ip.h:453-454`

```c
#define ip_cksum_carry(x) \
    (x = (x >> 16) + (x & 0xffff), (~(x + (x >> 16)) & 0xffff))
```

展开后：
```c
x = (x >> 16) + (x & 0xffff);      /* 第一次折叠：将高16位加到低16位 */
x = x + (x >> 16);                  /* 第二次折叠：处理可能的新进位 */
x = ~x & 0xffff;                    /* 取反码 */
return x;
```

### 10.4 完整计算流程

```
输入：数据缓冲区 buf，长度 len

1. 初始化：
   sum = 0

2. 累加16位字：
   for (i = 0; i < len/2; i++) {
       sum += *(uint16_t *)buf++;
   }

3. 处理奇数字节：
   if (len % 2 == 1) {
       sum += (uint16_t)(*(uint8_t *)buf) << 8;
   }

4. 折叠进位：
   while (sum >> 16) {
       sum = (sum & 0xFFFF) + (sum >> 16);
   }

5. 取反码：
   sum = ~sum;

6. 返回：
   return sum;
```

### 10.5 手动计算示例

假设计算数据：`0x00 0x01 0xF2 0xF3`

1. 按16位字分组：
   ```
   字1: 0x0001
   字2: 0xF2F3
   ```

2. 求和：
   ```
   0x0001
 + 0xF2F3
 --------
   0xF2F4
   ```

3. 折叠进位（如果需要）：
   ```
   0xF2F4
   ```
   （无需折叠，因为高16位为0）

4. 取反码：
   ```
   ~0xF2F4 = 0x0D0B
   ```

5. 最终校验和：`0x0D0B`

## 11. CRC32-C算法

### 11.1 CRC基础概念

CRC（Cyclic Redundancy Check，循环冗余校验）是一种基于多项式除法的错误检测算法：

- **多项式**：表示除数
- **消息**：被除数
- **余数**：校验值

### 11.2 CRC32-C多项式

```
CRC32-C多项式：x^32 + x^28 + x^27 + x^26 + x^25 + x^23 + x^22 +
             x^20 + x^19 + x^18 + x^14 + x^13 + x^11 + x^10 +
             x^9 + x^8 + x^6 + 1

十六进制表示：0x1EDC6F41
```

### 11.3 查表法实现

#### 11.3.1 预计算表

**文件位置**：`src/crc32ct.h:15-80`

```c
static unsigned long crc_c[256] = {
    0x00000000L, 0xF26B8303L, 0xE13B70F7L, 0x1350F3F4L,
    /* ... 256个值 ... */
};
```

每个值对应：`crc32(0x00000000, byte)`

#### 11.3.2 CRC32C宏

```c
#define CRC32C(c,d) (c=(c>>8)^crc_c[(c^(d))&0xFF])
```

参数：
- `c`：当前CRC值
- `d`：输入字节

操作：
1. 将当前CRC右移8位
2. 与查表值异或
3. 查表索引：`(c ^ d) & 0xFF`

### 11.4 完整计算过程

```c
unsigned long calculate_crc32c(const unsigned char *data, int len) {
    unsigned long crc = 0xFFFFFFFF;
    
    for (int i = 0; i < len; i++) {
        crc = (crc >> 8) ^ crc_c[(crc ^ data[i]) & 0xFF];
    }
    
    crc = ~crc;
    
    /* 字节序翻转 */
    return ((crc & 0xFF) << 24) |
           ((crc >> 8 & 0xFF) << 16) |
           ((crc >> 16 & 0xFF) << 8) |
           (crc >> 24 & 0xFF);
}
```

### 11.5 为什么需要字节序翻转？

CRC32-C标准规定输出为大端序，但：
- Intel x86是小端序
- 直接计算得到的是小端序
- 需要翻转字节序以符合标准

### 11.6 硬件加速

现代CPU支持CRC32指令（SSE4.2）：

```c
#ifdef __SSE4_2__
#include <smmintrin.h>

uint32_t crc32c_hardware(const uint8_t *data, size_t len) {
    uint32_t crc = 0xFFFFFFFF;
    
    /* 使用硬件指令 */
    for (size_t i = 0; i < len; i++) {
        crc = _mm_crc32_u8(crc, data[i]);
    }
    
    return ~crc;
}
#endif
```

性能对比：
- 查表法：~2 cycles/byte
- 硬件加速：~1 cycle/byte

## 12. 跨平台实现差异

### 12.1 字节序处理

#### 12.1.1 大端序（Big-Endian）

网络协议使用大端序（网络字节序）：

```
内存布局（大端序）：
地址 0: 0x12 (高字节)
地址 1: 0x34
地址 2: 0x56
地址 3: 0x78 (低字节)

表示值：0x12345678
```

#### 12.1.2 小端序（Little-Endian）

x86/x64使用小端序：

```
内存布局（小端序）：
地址 0: 0x78 (低字节)
地址 1: 0x56
地址 2: 0x34
地址 3: 0x12 (高字节)

表示值：0x12345678
```

### 12.4 对齐问题

某些平台对内存对齐有严格要求：


假设：

- 或者平台支持未对齐访问

## 13. 性能优化技巧

### 13.1 避免重复计算

```c
/* 错误：每次都重新计算 */
for (int i = 0; i < n; i++) {
    ip_checksum(buf[i], len, 0);
    /* 修改数据 */
    ip_checksum(buf[i], len, 0);  /* 重复计算！*/
}

/* 正确：只在修改后计算 */
for (int i = 0; i < n; i++) {
    ip_checksum(buf[i], len, 0);
}
```

```c
}
```

### 13.3 缓存友好访问

```c
/* 顺序访问，缓存友好 */
int sum_sequential(const uint16_t *data, int n) {
    int sum = 0;
    for (int i = 0; i < n; i++) {
        sum += data[i];
    }
    return sum;
}

/* 跳跃访问，缓存不友好 */
int sum_strided(const uint16_t *data, int n, int stride) {

    }
    return sum;
}
```

```c
#endif
```

### 15.1 发送自定义TCP SYN包

```c
#include <dnet.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int send_tcp_syn(const char *src_ip, const char *dst_ip,
                 uint16_t src_port, uint16_t dst_port) {
    ip_t *ip;
    uint8_t packet[64];
    struct ip_hdr *ip_hdr = (struct ip_hdr *)packet;
    struct tcp_hdr *tcp_hdr = (struct tcp_hdr *)(packet + 20);
    struct addr src, dst;
    
    /* 打开IP接口 */
    if ((ip = ip_open()) == NULL) {
        perror("ip_open");
        return -1;
    }
    
    /* 构造IP头部 */
    memset(ip_hdr, 0, sizeof(*ip_hdr));
    ip_hdr->ip_v = 4;
    ip_hdr->ip_hl = 5;
    ip_hdr->ip_tos = 0;
    ip_hdr->ip_len = htons(40);  /* IP头20 + TCP头20 */
    ip_hdr->ip_id = rand() & 0xffff;
    ip_hdr->ip_off = 0;
    ip_hdr->ip_ttl = 64;
    ip_hdr->ip_p = IP_PROTO_TCP;
    ip_hdr->ip_sum = 0;
    
    /* 转换IP地址 */
    addr_aton(src_ip, &src);
    addr_aton(dst_ip, &dst);
    ip_hdr->ip_src = src.addr_ip;
    ip_hdr->ip_dst = dst.addr_ip;
    
    /* 构造TCP头部 */
    memset(tcp_hdr, 0, sizeof(*tcp_hdr));
    tcp_hdr->th_sport = htons(src_port);
    tcp_hdr->th_dport = htons(dst_port);
    tcp_hdr->th_seq = htonl(rand());
    tcp_hdr->th_ack = 0;
    tcp_hdr->th_off = 5;
    tcp_hdr->th_flags = TH_SYN;
    tcp_hdr->th_win = htons(65535);
    tcp_hdr->th_urp = 0;
    tcp_hdr->th_sum = 0;
    
    /* 计算校验和 */
    ip_checksum(packet, 40, 0);
    
    /* 发送数据包 */
    if (ip_send(ip, packet, 40) < 0) {
        perror("ip_send");
        ip_close(ip);
        return -1;
    }
    
    printf("Sent TCP SYN: %s:%d -> %s:%d\n",
           src_ip, src_port, dst_ip, dst_port);
    printf("  IP checksum: 0x%04x\n", ntohs(ip_hdr->ip_sum));
    printf("  TCP checksum: 0x%04x\n", ntohs(tcp_hdr->th_sum));
    
    ip_close(ip);
    return 0;
}

int main(int argc, char *argv[]) {
    if (argc != 5) {
        fprintf(stderr, "Usage: %s <src_ip> <dst_ip> <src_port> <dst_port>\n",
                argv[0]);
        return 1;
    }
    
    srand(time(NULL));
    return send_tcp_syn(argv[1], argv[2], atoi(argv[3]), atoi(argv[4]));
}
```

### 15.2 发送ICMP Echo请求（Ping）

```c
#include <dnet.h>
#include <stdio.h>
#include <stdlib.h>

int send_ping(const char *dst_ip) {
    ip_t *ip;
    uint8_t packet[128];
    struct ip_hdr *ip_hdr = (struct ip_hdr *)packet;
    struct icmp_hdr *icmp_hdr = (struct icmp_hdr *)(packet + 20);
    struct icmp_msg_echo *echo = (struct icmp_msg_echo *)(packet + 24);

    if ((ip = ip_open()) == NULL) {

        return -1;

    ip_hdr->ip_v = 4;
    ip_hdr->ip_hl = 5;
    ip_hdr->ip_len = htons(28);  /* IP头20 + ICMP头4 + Echo数据8 */
    ip_hdr->ip_id = rand() & 0xffff;
    ip_hdr->ip_ttl = 64;
    ip_hdr->ip_p = IP_PROTO_ICMP;
    ip_hdr->ip_src = inet_addr("0.0.0.0");  /* 由系统填充 */

    ip_hdr->ip_dst = dst.addr_ip;

    icmp_hdr->icmp_type = ICMP_ECHO;
    icmp_hdr->icmp_code = 0;
    icmp_hdr->icmp_cksum = 0;
    
    echo->icmp_id = htons(getpid());
    echo->icmp_seq = htons(1);
    memcpy(echo->icmp_data, "Hello", 5);
    
    /* 计算校验和 */
    icmp_checksum(icmp_hdr, 12);  /* ICMP头4 + 数据8 */
    ip_checksum(packet, 28, 0);
    
    /* 发送数据包 */
    if (ip_send(ip, packet, 28) < 0) {
        perror("ip_send");
        ip_close(ip);
        return -1;
    }
    
    printf("Sent ICMP Echo to %s\n", dst_ip);
    printf("  IP checksum: 0x%04x\n", ntohs(ip_hdr->ip_sum));
    printf("  ICMP checksum: 0x%04x\n", ntohs(icmp_hdr->icmp_cksum));
    
    ip_close(ip);
    return 0;
}
```

### 15.3 验证接收到的数据包

```c
#include <dnet.h>
#include <stdio.h>
#include <string.h>

int verify_ip_packet(const uint8_t *packet, size_t len) {
    struct ip_hdr *ip_hdr = (struct ip_hdr *)packet;
    uint16_t original_sum;
    int valid = 1;
    
    /* 检查最小长度 */
    if (len < IP_HDR_LEN) {
        printf("Packet too short: %zu bytes\n", len);
        return 0;
    }
    
    /* 验证IP头部校验和 */
    original_sum = ip_hdr->ip_sum;
    ip_hdr->ip_sum = 0;
    ip_checksum((void *)packet, len, 0);
    
    if (original_sum != ip_hdr->ip_sum) {
        printf("IP checksum mismatch:\n");
        printf("  Expected: 0x%04x\n", ntohs(original_sum));
        printf("  Calculated: 0x%04x\n", ntohs(ip_hdr->ip_sum));
        valid = 0;
    }
    
    /* 恢复原始值 */
    ip_hdr->ip_sum = original_sum;
    
    /* 根据协议验证上层校验和 */
    int ip_hl = ip_hdr->ip_hl << 2;
    int payload_len = ntohs(ip_hdr->ip_len) - ip_hl;
    uint8_t *payload = (uint8_t *)packet + ip_hl;
    
    switch (ip_hdr->ip_p) {
    case IP_PROTO_TCP: {
        struct tcp_hdr *tcp_hdr = (struct tcp_hdr *)payload;
        uint16_t tcp_sum = tcp_hdr->th_sum;
        
        tcp_hdr->th_sum = 0;
        tcp_checksum(ip_hdr, tcp_hdr, payload_len);
        
        if (tcp_sum != tcp_hdr->th_sum) {
            printf("TCP checksum mismatch\n");
            valid = 0;
        }
        tcp_hdr->th_sum = tcp_sum;
        break;
    }
    case IP_PROTO_UDP: {
        struct udp_hdr *udp_hdr = (struct udp_hdr *)payload;
        
        if (udp_hdr->uh_sum != 0) {
            uint16_t udp_sum = udp_hdr->uh_sum;
            
            udp_hdr->uh_sum = 0;
            udp_checksum(ip_hdr, udp_hdr, payload_len);
            
            if (udp_sum != udp_hdr->uh_sum) {
                printf("UDP checksum mismatch\n");
                valid = 0;
            }
            udp_hdr->uh_sum = udp_sum;
        }
        break;
    }
    case IP_PROTO_ICMP: {
        struct icmp_hdr *icmp_hdr = (struct icmp_hdr *)payload;
        uint16_t icmp_sum = icmp_hdr->icmp_cksum;
        
        icmp_hdr->icmp_cksum = 0;
        icmp_checksum(icmp_hdr, payload_len);
        
        if (icmp_sum != icmp_hdr->icmp_cksum) {
            printf("ICMP checksum mismatch\n");
            valid = 0;
        }
        icmp_hdr->icmp_cksum = icmp_sum;
        break;
    }
    }
    
    return valid;
}
```
