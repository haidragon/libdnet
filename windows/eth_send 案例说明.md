# libdnet Windows 以太网帧发送案例说明

## 案例名称
以太网帧发送 (eth_send)

## 功能描述
本案例演示如何通过指定的网络接口发送原始的以太网数据帧。这是最底层的网络通信方式，可以直接控制以太网帧的各个字段。

## 技术要点

### 1. 以太网帧结构
```
+----------------+----------------+------------+------------------+
| 目标 MAC(6B)   | 源 MAC(6B)     | 类型 (2B)  | 数据 (46-1500B)  |
+----------------+----------------+------------+------------------+
```

### 2. 核心 API
- `eth_open()`: 打开指定的网络接口
- `eth_get_addr()`: 获取接口的硬件 MAC 地址
- `eth_send()`: 发送原始以太网帧
- `eth_close()`: 关闭以太网接口

### 3. 关键步骤
1. 打开以太网接口
2. 获取源 MAC 地址
3. 解析目标 MAC 地址
4. 构造以太网帧（包括目标 MAC、源 MAC、类型、数据）
5. 发送数据帧
6. 关闭接口

## 编译方法

```bash
gcc -I../include -L../src/.libs -o eth_send.exe eth_send.c -ldnet -lws2_32 -liphlpapi
```

**依赖库说明：**
- `-ldnet`: libdnet 库
- `-lws2_32`: Windows Socket 2 库
- `-liphlpapi`: IP Helper API 库

## 运行方法

### 基本用法
```bash
eth_send.exe <网络接口名> <目标 MAC 地址>
```

### 示例
```bash
# 查看可用的网络接口（使用 example.exe）
example.exe

# 发送到指定 MAC 地址
eth_send.exe "Local Area Connection" 00:11:22:33:44:55

# 使用无线网卡接口
eth_send.exe "Wireless Network Connection" ff:ff:ff:ff:ff:ff
```

## 参数说明

### 网络接口名
Windows 系统上的网络接口名称，例如：
- `"Local Area Connection"` - 有线网卡
- `"Wireless Network Connection"` - 无线网卡
- `"Ethernet"` - Windows 10/11 的默认有线网卡名
- `"Wi-Fi"` - Windows 10/11 的默认无线网卡名

可以使用 `example.exe` 程序查看所有可用接口。

### 目标 MAC 地址
标准的 MAC 地址格式，支持以下分隔符：
- `00:11:22:33:44:55` (冒号分隔)
- `00-11-22-33-44-55` (短横线分隔)
- `0011.2233.4455` (点分隔，Cisco 格式)

特殊地址：
- `ff:ff:ff:ff:ff:ff` - 广播地址
- `01:00:5e:xx:xx:xx` - 组播地址

## 注意事项

### 1. 权限要求
发送原始以太网帧需要管理员权限：
- 在 Windows 上以管理员身份运行命令行
- 可能需要安装 WinPcap/Npcap 驱动

### 2. 最小帧长度
以太网规定最小帧长度为 64 字节（不包括前导码和帧校验序列）：
- 如果数据部分不足 46 字节，需要填充
- 本案例自动填充到 64 字节

### 3. 网络环境限制
- 只能发送到同一局域网内的设备
- 交换机可能会过滤非法 MAC 地址的帧
- 某些网卡可能不支持自定义以太网类型

### 4. 以太网类型值
示例中使用 `0x1234` 作为测试类型，实际应用中应使用标准类型：
- `0x0800` - IPv4
- `0x0806` - ARP
- `0x86DD` - IPv6
- `0x8100` - VLAN Tagged Frame

## 扩展练习

### 1. 修改以太网类型
将代码中的类型改为 IPv4：
```c
frame[12] = 0x08;
frame[13] = 0x00;
```

### 2. 发送 ARP 请求
构造一个 ARP 请求帧，类型设置为 `0x0806`。

### 3. 批量发送
修改程序支持批量发送多个帧，用于压力测试。

### 4. 添加命令行选项
使用 getopt 或类似库添加更多选项：
- `-i` 指定接口
- `-d` 指定目标 MAC
- `-t` 指定类型
- `-p` 指定有效载荷文件

## 故障排查

### 问题 1: 无法打开以太网接口
**原因**: 
- 接口名称错误
- 没有管理员权限
- 缺少 WinPcap/Npcap 驱动

**解决方法**:
- 使用 `example.exe` 查看正确的接口名称
- 以管理员身份运行
- 安装 Npcap: https://nmap.org/npcap/

### 问题 2: 发送失败
**原因**:
- 网络连接断开
- 防火墙阻止
- 网卡不支持

**解决方法**:
- 检查网络连接状态
- 临时关闭防火墙测试
- 尝试其他网卡

### 问题 3: 接收方收不到帧
**原因**:
- 目标 MAC 不存在
- 交换机过滤
- VLAN 隔离

**解决方法**:
- 使用广播地址测试
- 检查交换机配置
- 确保在同一 VLAN
