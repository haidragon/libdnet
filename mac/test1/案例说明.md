# libdnet macOS 案例说明文档

> 基于 libdnet-1.13 的 macOS 平台网络编程案例

---

## 项目概述

本目录包含两个基于 libdnet 的实用示例程序，展示了如何使用 libdnet 进行网络编程：

1. **interface_scanner.c** - 网络接口扫描工具
2. **packet_builder.c** - 数据包构建工具

这些案例程序演示了 libdnet 的核心 API 使用方法，包括：
- 网络接口扫描和信息获取
- 路由表和 ARP 缓存查询
- 各种网络协议数据包的构建（ICMP、TCP、UDP、ARP）
- 地址操作和转换

## 前置条件

### 1. 系统要求

- macOS 10.12 或更高版本
- Xcode Command Line Tools
- 管理员权限（用于某些操作）

### 2. 安装编译工具

```bash
# 安装 Xcode Command Line Tools
xcode-select --install

# 验证安装
gcc --version
make --version
```

### 3. 编译 libdnet

在编译示例程序之前，需要先编译 libdnet 库：

```bash
cd /path/to/libdnet-1.13

# 配置
./configure

# 编译
make

# 安装（可选，推荐）
sudo make install
```

安装后，头文件和库文件会被安装到系统目录：
- 头文件: `/usr/local/include/dnet/`
- 库文件: `/usr/local/lib/libdnet.*`

## 编译示例程序

### 方法 1: 系统已安装 libdnet

如果已经运行 `sudo make install`：

```bash
cd mac

# 编译所有程序
make

# 或单独编译
gcc -Wall -Wextra -O2 -g -o interface_scanner interface_scanner.c -ldnet
gcc -Wall -Wextra -O2 -g -o packet_builder packet_builder.c -ldnet
```

### 方法 2: 使用源码目录中的库

如果不想安装到系统目录：

```bash
cd mac

# 编译 interface_scanner
gcc -Wall -Wextra -O2 -g \
    -I../include \
    -L../src/.libs \
    -o interface_scanner interface_scanner.c -ldnet

# 编译 packet_builder
gcc -Wall -Wextra -O2 -g \
    -I../include \
    -L../src/.libs \
    -o packet_builder packet_builder.c -ldnet

# 运行时可能需要设置库路径
export DYLD_LIBRARY_PATH=../src/.libs:$DYLD_LIBRARY_PATH
./interface_scanner
```

### 方法 3: 静态链接

完全避免动态库依赖问题：

```bash
cd mac

# 编译 interface_scanner（静态链接）
gcc -Wall -Wextra -O2 -g \
    -I../include \
    ../src/.libs/libdnet.a \
    -o interface_scanner interface_scanner.c

# 编译 packet_builder（静态链接）
gcc -Wall -Wextra -O2 -g \
    -I../include \
    ../src/.libs/libdnet.a \
    -o packet_builder packet_builder.c
```

## 代码解析

### 1. interface_scanner.c 代码结构

#### 核心数据结构

```c
// 回调函数用于遍历接口
static int print_interface(const struct intf_entry *entry, void *arg) {
    // entry 包含接口的所有信息
    printf("接口: %s\n", entry->intf_name);
    printf("IP: %s/%d\n", ip_str, entry->intf_addr.addr_bits);
    printf("MAC: %s\n", mac_str);
    return 0;
}
```

#### 接口扫描流程

```c
// 1. 打开接口句柄
intf_t *i = intf_open();

// 2. 遍历所有接口
int count = 0;
intf_loop(i, print_interface, &count);

// 3. 关闭句柄
intf_close(i);
```

#### 地址操作示例

```c
struct addr addr1, addr2, net, bcast;

// 字符串 → 地址
addr_pton("192.168.1.100/24", &addr1);

// 计算网络地址
addr_net(&addr1, &net);

// 计算广播地址
addr_bcast(&addr1, &bcast);

// 比较地址
int result = addr_cmp(&addr1, &addr2);
```

### 2. packet_builder.c 代码结构

#### ICMP 数据包构建

```c
void build_icmp_ping(uint8_t *packet, size_t *packet_len,
                     ip_addr_t src_ip, ip_addr_t dst_ip,
                     uint16_t icmp_id, uint16_t icmp_seq,
                     const char *data, size_t data_len) {
    // 1. 构建 IP 头部
    ip_pack_hdr(packet, IP_TOS_DEFAULT, total_len,
                1234, 0, 64, IP_PROTO_ICMP, src_ip, dst_ip);

    // 2. 构建 ICMP 头部
    struct icmp_hdr *icmp = (struct icmp_hdr *)(packet + IP_HDR_LEN);
    icmp_pack_hdr_echo(icmp, ICMP_ECHO, ICMP_CODE_NONE,
                       icmp_id, icmp_seq, data, data_len);

    // 3. 计算校验和
    ip_checksum(packet, IP_HDR_LEN, 0);
    icmp_checksum(icmp, icmp_len);
}
```

#### TCP 数据包构建

```c
void build_tcp_syn(uint8_t *packet, size_t *packet_len,
                   ip_addr_t src_ip, uint16_t src_port,
                   ip_addr_t dst_ip, uint16_t dst_port,
                   uint32_t seq, uint32_t ack,
                   uint8_t flags, uint16_t win) {
    // 1. 构建 IP 头部
    ip_pack_hdr(packet, IP_TOS_DEFAULT, total_len,
                1234, 0, 64, IP_PROTO_TCP, src_ip, dst_ip);

    // 2. 构建 TCP 头部
    tcp_pack_hdr(packet + IP_HDR_LEN, src_port, dst_port,
                 seq, ack, flags, win, 0);

    // 3. 计算校验和
    ip_checksum(packet, IP_HDR_LEN, 0);

    struct ip_hdr *ip = (struct ip_hdr *)packet;
    struct tcp_hdr *tcp = (struct tcp_hdr *)(packet + IP_HDR_LEN);
    tcp_checksum(ip, tcp, TCP_HDR_LEN);
}
```

#### ARP 数据包构建

```c
void build_arp_request(uint8_t *packet, size_t *packet_len,
                       eth_addr_t src_mac, ip_addr_t src_ip,
                       ip_addr_t dst_ip) {
    eth_addr_t broadcast_mac;
    eth_pton(ETH_ADDR_BROADCAST, &broadcast_mac);

    // 1. 构建以太网头部
    eth_pack_hdr(packet, broadcast_mac, src_mac, ETH_TYPE_ARP);

    // 2. 构建 ARP 头部
    arp_pack_hdr_ethip(packet + ETH_HDR_LEN,
                       ARP_OP_REQUEST,
                       src_mac, src_ip,
                       broadcast_mac, dst_ip);
}
```

## API 参考

### 接口操作 (intf.h)

| 函数 | 说明 | 返回值 |
|------|------|--------|
| `intf_open()` | 打开接口句柄 | intf_t* 成功/NULL 失败 |
| `intf_get()` | 获取指定接口 | 0 成功/-1 失败 |
| `intf_get_src()` | 根据源地址获取接口 | 0 成功/-1 失败 |
| `intf_loop()` | 遍历所有接口 | 0 成功/-1 失败 |
| `intf_close()` | 关闭句柄 | intf_t* |

### 地址操作 (addr.h)

| 函数 | 说明 |
|------|------|
| `addr_pton(src, dst)` | 文本地址 → 二进制结构 |
| `addr_ntop(src, dst, size)` | 二进制结构 → 文本地址 |
| `addr_cmp(a, b)` | 比较两个地址 |
| `addr_net(a, b)` | 计算网络地址 |
| `addr_bcast(a, b)` | 计算广播地址 |

### 路由操作 (route.h)

| 函数 | 说明 |
|------|------|
| `route_open()` | 打开路由句柄 |
| `route_loop(callback, arg)` | 遍历路由表 |
| `route_close()` | 关闭句柄 |

### ARP 操作 (arp.h)

| 函数 | 说明 |
|------|------|
| `arp_open()` | 打开 ARP 句柄 |
| `arp_loop(callback, arg)` | 遍历 ARP 缓存 |
| `arp_close()` | 关闭句柄 |

### 以太网操作 (eth.h)

| 函数 | 说明 |
|------|------|
| `eth_pton(src, dst)` | 文本 MAC → 二进制 |
| `eth_ntop(eth, dst, len)` | 二进制 MAC → 文本 |
| `eth_pack_hdr(h, dst, src, type)` | 构建以太网头部 |

### IP 操作 (ip.h)

| 函数 | 说明 |
|------|------|
| `ip_pack_hdr(buf, ...)` | 构建 IP 头部 |
| `ip_checksum(buf, len, flags)` | 计算 IP 校验和 |
| `ip_open()` | 打开 IP 句柄（发送数据包） |
| `ip_send(i, buf, len)` | 发送 IP 数据包 |
| `ip_close(i)` | 关闭句柄 |

### TCP 操作 (tcp.h)

| 函数 | 说明 |
|------|------|
| `tcp_pack_hdr(buf, ...)` | 构建 TCP 头部 |
| `tcp_checksum(ip, tcp, len)` | 计算 TCP 校验和 |

### UDP 操作 (udp.h)

| 函数 | 说明 |
|------|------|
| `udp_pack_hdr(hdr, ...)` | 构建 UDP 头部 |

### ICMP 操作 (icmp.h)

| 函数 | 说明 |
|------|------|
| `icmp_pack_hdr_echo(...)` | 构建 ICMP Echo 头部 |
| `icmp_checksum(icmp, len)` | 计算 ICMP 校验和 |

## 常见问题解决

### 1. 编译错误: 找不到头文件

**错误**: `fatal error: dnet.h: No such file or directory`

**解决方案**:

```bash
# 方法 1: 指定头文件路径
gcc -I/path/to/libdnet/include -o test test.c

# 方法 2: 安装到系统
cd /path/to/libdnet
sudo make install
```

### 2. 链接错误: 找不到 libdnet

**错误**: `ld: library not found for -ldnet`

**解决方案**:

```bash
# 方法 1: 指定库路径
gcc -L/path/to/libdnet/.libs -o test test.c -ldnet

# 方法 2: 设置环境变量
export DYLD_LIBRARY_PATH=/path/to/libdnet/.libs:$DYLD_LIBRARY_PATH

# 方法 3: 静态链接
gcc /path/to/libdnet/.libs/libdnet.a -o test test.c
```

### 3. 运行时错误: 权限不足

**错误**: `Operation not permitted` 或 `Permission denied`

**原因**: 某些操作需要管理员权限

**解决方案**:

```bash
sudo ./interface_scanner
sudo ./packet_builder
```

### 4. 函数返回 NULL 或 -1

**检查方法**:

```c
intf_t *i = intf_open();
if (i == NULL) {
    perror("intf_open");
    // 检查 errno
    // EACCES - 权限不足
    // ENOENT - 设备不存在
    return -1;
}
```

## 与 Mettle 的集成

这些示例程序展示了 Mettle 如何使用 libdnet 进行网络操作：

### Mettle 使用场景

```c
// 在 Mettle 中获取网络接口
void mettle_init_network(void) {
    intf_t *i = intf_open();

    // 遍历接口，记录可用的网络接口
    intf_loop(i, mettle_collect_interfaces, NULL);

    // 获取路由信息
    route_t *r = route_open();
    route_loop(r, mettle_collect_routes, NULL);
    route_close(r);

    intf_close(i);
}

// Mettle 发送数据包
void mettle_send_raw_packet(void *data, size_t len) {
    ip_t *ip = ip_open();
    ip_send(ip, data, len);
    ip_close(ip);
}
```
