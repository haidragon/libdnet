# libdnet 网络扫描示例 (test21)

## 功能说明

本示例展示如何使用 libdnet 库进行基本的网络扫描，包括：
- ARP 缓存查询
- 路由表扫描
- 接口扫描
- 端口扫描概念
- 扫描建议

## 编译方法

```bash
make
```

## 运行方法

```bash
make run
```

或者直接运行：

```bash
./network_scan
```

## 扫描类型

### 1. ARP 扫描
- 发送 ARP 请求到本地网络
- 检测响应的主机
- 获取 MAC 地址
- 仅适用于同一网段

### 2. 路由表扫描
- 遍历系统路由表
- 发现可达网络
- 识别网关
- 了解网络拓扑

### 3. 接口扫描
- 列举所有网络接口
- 检查接口状态
- 获取 IP 地址
- 识别活动接口

### 4. 端口扫描（概念）
- TCP Connect: 完整 TCP 连接
- TCP SYN: 半连接扫描
- UDP: UDP 端口探测
- 需要原始 socket 或标准 socket

## 扫描建议

### 合规性
- 仅在授权网络中使用
- 获取必要的许可
- 遵守法律法规
- 尊重隐私

### 技术建议
- 控制扫描速率
- 使用适当的超时
- 处理异常情况
- 记录扫描结果

### 工具推荐
对于完整扫描，建议使用专业工具：
- nmap: 最流行的网络扫描工具
- masscan: 高速端口扫描器
- zmap: 互联网级扫描器

## API 说明

### arp_open()
打开 ARP 缓存。

```c
arp_t *arp_open(void);
```

### arp_get()
查询 ARP 条目。

```c
int arp_get(arp_t *arp, struct arp_entry *entry);
```

### route_open()
打开路由表。

```c
route_t *route_open(void);
```

### route_loop()
遍历路由表。

```c
int route_loop(route_t *rt, struct route_entry *entry);
```

### intf_open()
打开接口操作。

```c
int_t *intf_open(void);
```

### intf_loop()
遍历网络接口。

```c
int intf_loop(int_t *iop, struct intf_entry *entry,
              int (*callback)(int_t *, const struct intf_entry *));
```

## 注意事项

1. **法律合规**: 仅在授权网络中使用
2. **权限**: 某些操作需要 root 权限
3. **网络影响**: 控制扫描速率
4. **准确性**: 扫描结果可能不完整
5. **隐私**: 尊重他人隐私

## 实现完整扫描

要实现完整的网络扫描，需要：

1. **ARP 扫描**:
   - 遍历 IP 范围
   - 发送 ARP 请求
   - 等待响应
   - 记录结果

2. **Ping 扫描**:
   - 发送 ICMP Echo
   - 等待响应
   - 识别活动主机

3. **端口扫描**:
   - 针对特定端口
   - 发送探测包
   - 分析响应

4. **服务识别**:
   - 分析端口响应
   - 识别服务类型
   - 获取版本信息
